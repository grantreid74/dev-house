**How Dev-House delivers to customers.** Select the model before starting any customer engagement.

---

## Selection Guide

**Default: D1 (Handover)**

Unless the customer explicitly requests operational management AND offers a retainer that covers that ongoing cost, use D1. Dev-House is currently 2 people. Managing other people's operations is a different business.

**Consider D2 only when:**
- Customer offers a retainer that covers ongoing operational overhead
- Customer has no internal ops capability and the engagement is long-term
- The scope has been explicitly agreed and priced — not assumed

---

## What These Are Not

- **Generation Infrastructure** (G-series) — where Dev-House runs its agents to generate code. See `docs/deployment/models/`
- **Customer Deployment Patterns** (D-T3, D-T4, D-BYOC) — what the customer's system looks like. See `docs/deployment/deployment-patterns.md`

Delivery model and deployment pattern are chosen independently. A D1 handover customer can receive a D-T4 (full isolation) deployment pattern. The delivery model is about who operates it; the deployment pattern is about what is built.
